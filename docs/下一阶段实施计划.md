# 下一阶段实施计划（Phase 2）

> 目标：Agent 编排 + 结构化存储 + 内容评分，形成可追溯的端到端流水线。

---

## 一、阶段目标

1. 引入 LangGraph（或 lightweight 自研 StateGraph）串联 Trend → Strategy → Script。
2. 建立 SQLite/PostgreSQL 结构化存储（trends、scripts、videos、agents_log）。
3. 实现内容评分 Agent（创意、营销价值、传播潜力）。
4. 将现有 run_all 流程与 Agent 编排、DB 打通，输出可入库、可追溯。

---

## 二、任务清单

### 任务 2.1：引入 LangGraph 与工作流骨架

| 项目 | 内容 |
|------|------|
| **依赖** | `langgraph>=0.2.0`、`langchain-core` |
| **目录** | `src/workflow/` |
| **产出** | `workflow/graph.py`：定义 `StateGraph`，节点 `trend`、`strategy`、`script` |
| **状态** | 每个节点为 callable，接收共享 `state`，返回更新后的 `state` |

**验收**：`python -c "from src.workflow.graph import app; print(app)"` 可执行，无导入错误。

---

### 任务 2.2：统一 Agent 接口与 State 定义

| 项目 | 内容 |
|------|------|
| **文件** | `src/workflow/state.py`、`src/agents/base.py` |
| **State 字段** | `trend_data`、`strategy_output`、`script_output`、`scores`、`errors` |
| **Agent 基类** | `BaseAgent.run(state: dict) -> dict`，返回 state 增量 |

**验收**：`TrendAgent`、`StrategyAgent`、`ScriptAgent` 可被工作流节点调用。

---

### 任务 2.3：数据库模型与迁移

| 项目 | 内容 |
|------|------|
| **存储** | 先 SQLite（`data/yingxiao.db`），预留 PostgreSQL 切换 |
| **表** | `trends`、`scripts`、`videos`、`agents_log` |
| **工具** | SQLAlchemy 或纯 sqlite3，在 `src/db/` 下实现 |
| **脚本** | `scripts/init_db.py` 创建表 |

**表结构**：

- `trends`：id, platform, video_url, likes, comments, views, topic, created_at
- `scripts`：id, title, hook, script, scene, industry, trend_id, created_at
- `videos`：id, script_id, video_url, score, created_at
- `agents_log`：id, agent_name, task, result, time

**验收**：`python scripts/init_db.py` 可创建表，无报错。

---

### 任务 2.4：趋势/策略/脚本 Agent 入库

| 项目 | 内容 |
|------|------|
| **趋势 Agent** | 输出写入 `trends`，返回 `trend_ids` 供下游使用 |
| **策略 Agent** | 接收 `trend_data`，输出写入 `scripts` 的 title/hook 等 |
| **脚本 Agent** | 接收 `strategy_output`，补全 script/scene，写入 `scripts` |
| **日志** | 每次 Agent 调用写入 `agents_log` |

**验收**：跑完整工作流后，`trends`、`scripts`、`agents_log` 有新增记录。

---

### 任务 2.5：内容评分 Agent

| 项目 | 内容 |
|------|------|
| **文件** | `src/agents/score_agent.py` |
| **输入** | 脚本内容（title、hook、script） |
| **输出** | 创意评分、营销价值、传播潜力、综合 score (0–100) |
| **实现** | 调用 LLM，按预设 prompt 打分，解析 JSON 输出 |

**验收**：给定脚本，`ScoreAgent.run(script)` 返回含 `creative_score`、`marketing_value`、`spread_potential`、`score` 的 dict。

---

### 任务 2.6：集成到 run_all 与看板

| 项目 | 内容 |
|------|------|
| **run_all** | 在 AI 分析后，可选执行工作流 `app.invoke(initial_state)`，结果入库 |
| **看板** | 新增「内容评分」或扩展现有页面，展示 scripts 及对应 score |
| **环境变量** | `USE_WORKFLOW=true` 时启用编排；`DB_PATH` 指定 SQLite 路径 |

**验收**：`python run_all.py` 后，数据库中有 trends、scripts、agents_log；看板可展示评分。
