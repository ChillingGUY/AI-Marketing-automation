## 一、概述

### 1.1 平台简介

AI 营销自动化平台是一套面向短视频营销的数据分析与创意辅助系统。系统自动抓取热门视频数据，通过 AI 分析提取爆款规律，并支持创意生成、知识库沉淀与自动化推送，帮助企业减少重复工作、提升创意产出与营销效率。

**设计目标**：把内容策划、脚本生成、视频制作、趋势分析等流程自动化，并通过企业协作平台（如飞书）分发给员工使用。

**相关文档**：
- [**实战使用文档.md**](实战使用文档.md) — **从零到功能演示的完整操作指南（推荐）**
- [設計方案_AI內容自動化.md](設計方案_AI內容自動化.md) — 详细架构与 8 大 Agent 设计
- [蓝图.md](蓝图.md) — 分阶段路线图与技术架构
- [下一阶段实施计划.md](下一阶段实施计划.md) — Phase 2 任务清单与实施顺序

### 1.2 核心功能

| 功能模块 | 说明 |
|----------|------|
| **数据采集** | 从 TikTok 获取热门视频（点赞、播放、评论、标签等），支持 CSV 导入 |
| **AI 分析** | 热门视频拆解、爆款结构提取、创意生成、内容脚本生成 |
| **热门趋势看板** | 可视化展示今日热门榜、趋势概览、时长分布 |
| **创意生成工具** | 输入主题生成 AI 创意与脚本，可选员工风格匹配 |
| **员工画像系统** | 员工数字人画像、自定义 prompt、创意历史 |
| **Agent 与 RAG** | Agent 对话、RAG 检索、数据分析 |
| **爆款内容库** | 存储与检索爆款模型、热门拆解，构建营销知识库 |
| **自动化推送** | 飞书机器人自动日报、创意推送 |

### 1.3 系统架构

```
数据来源层 → 数据处理层 → AI 分析层 → 营销平台 / 自动化系统
(TikTok/CSV)  (ETL)      (LLM)       (看板/创意/飞书)
```

### 1.4 LLM 应用四大支柱

| 支柱 | 实现 |
|------|------|
| **Agent 系统** | 营销 Agent，工具：热门榜、爆款模型、RAG 检索、员工列表；支持任务编排 |
| **RAG 知识库** | ChromaDB 向量存储，爆款模型/创意拆解入库，检索增强创意生成 |
| **数据分析** | ETL、趋势看板、Agent 驱动的趋势分析报告 |
| **自动化** | 定时任务、飞书推送、完整流水线编排 |

## 二、安装与配置

### 2.1 环境要求

- Python 3.9 及以上
- Windows / macOS / Linux

### 2.2 安装步骤

1. **进入项目目录**
   ```bash
   cd d:\yingxiao
   ```

2. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

3. **配置环境变量**（可选，可跳过）
   - 复制 `.env.example` 为 `.env`
   - 按需填写以下变量（见 2.3）

> **快速 DEMO 测试：可跳过 2.3 配置步骤**  
> 不创建 `.env` 也能运行完整流程。系统会：  
> - 使用内置示例数据（`data/raw/sample_tiktok_hot.csv`）替代 TikTok 抓取  
> - 使用规则模板生成创意（效果简化，但流程完整）  
> - 跳过飞书推送  
> 直接执行 `python run_all.py` 或 `streamlit run dashboard.py` 即可体验。
>
> **注意**：DEMO 数据的播放量/点赞为模拟值，无真实视频链接。  
> **要获取真实可播放的今日热门视频**，必须配置 `TIKHUB_API_TOKEN` 后运行 `python run_all.py`。

### 2.3 LLM 配置（通义 / OpenAI）

支持 **通义千问**（推荐国内）与 **OpenAI**，配置任一即可：

| 变量 | 说明 |
|------|------|
| `DASHSCOPE_API_KEY` | 通义/百炼 API Key（阿里云控制台获取） |
| `DASHSCOPE_MODEL` | 模型，默认 `qwen-plus` |
| `OPENAI_API_KEY` | OpenAI 或兼容接口 Key |
| `LLM_PROVIDER` | 指定 `dashscope` 或 `openai`，空则自动检测 |

提示词已规范为**营销设计专业领域**：自然语言输出、专业术语（卖点、痛点、CTA、完播率等）、商业价值导向、文案字数适中（40-120字/段）。

### 2.4 环境变量配置（可选）

| 变量名 | 必填 | 说明 | 获取方式 |
|--------|------|------|----------|
| `TIKHUB_API_TOKEN` | 否* | TikTok 数据抓取 | 注册 [TikHub](https://tikhub.io)，在用户中心创建 API Token |
| `DASHSCOPE_API_KEY` | 否 | 通义千问 LLM | 阿里云百炼控制台 |
| `OPENAI_API_KEY` | 否 | OpenAI LLM | OpenAI 控制台或兼容接口 |
| `OPENAI_BASE_URL` | 否 | 自定义 LLM 接口地址 | 使用 Azure / 本地模型时填写 |
| `LLM_MODEL` | 否 | OpenAI 模型名称 | 默认 `gpt-4o-mini` |
| `FEISHU_APP_ID` | 否 | 飞书应用 ID | 飞书开放平台创建应用 |
| `FEISHU_APP_SECRET` | 否 | 飞书应用密钥 | 同上 |
| `FEISHU_CHAT_ID` | 否 | 群聊 ID | 将机器人拉入群后获取 |

\* 未配置 TikHub Token 时，将使用示例数据演示流程。

### 2.5 中国大陆用户

如需访问 TikHub，可将 API 基础地址改为：
```
TIKHUB_API_BASE=https://api.tikhub.dev
```

## 三、使用指南

### 3.1 一键执行完整流程

**命令：**
```bash
python run_all.py
```

> 无需配置 `.env` 即可运行，会自动使用示例数据完成 DEMO。

**执行顺序：**
1. 数据采集（TikTok 热门视频或 CSV 示例）
2. ETL 处理与保存
3. AI 分析（热门拆解、爆款结构、创意生成）
4. 同步至爆款内容库
5. 飞书推送（若已配置）

**输出：**
- 原始数据：`data/raw/tiktok_hot_YYYYMMDD_HHMMSS.csv`
- 处理后数据：`data/processed/hot_rank_YYYYMMDD_HHMMSS.csv`
- AI 分析结果：`data/processed/ai_analysis_YYYYMMDD_HHMMSS.json`
- 知识库：`data/knowledge/` 下自动更新

### 3.2 单独执行数据采集

**命令：**
```bash
python main.py
```

用于仅执行「抓取 + ETL + AI 分析」，不包含飞书推送。控制台会输出今日热门视频榜 Top 10。

### 3.3 营销平台（Web 界面）

**启动：**
```bash
streamlit run dashboard.py
```

浏览器会自动打开（默认 `http://localhost:8501`）。

**页面说明：**

| 页面 | 功能 |
|------|------|
| **主页** | 今日热门视频榜、趋势概览、时长分布、AI 分析摘要 |
| **创意生成工具** | 输入主题生成创意与脚本，可选员工风格、RAG 增强 |
| **爆款内容库** | 同步分析结果、按标签检索、查看历史爆款模型 |
| **员工画像系统** | 员工数字人管理、创意历史 |
| **Agent 与 RAG** | Agent 对话、RAG 检索、数据分析报告 |

**操作示例：**

- **创意生成**：在「创意生成工具」选择员工数字人（如搞笑型创意策划），输入主题后生成，内容将匹配该员工风格。
- **员工画像**：在「员工画像系统」新增员工，填写岗位、内容风格、AI助手类型、擅长领域，可自定义 prompt 模板。
- **爆款检索**：在「爆款内容库」输入「开箱,测评」，点击「检索」查看相关拆解。

**Agent 与 RAG：**

- **Agent 对话**：输入任务（如「分析今日热门趋势」），Agent 自动调用工具并回答
- **RAG 知识库**：从爆款内容库同步后支持语义检索，创意生成时可勾选「使用 RAG 知识库增强」
- **数据分析**：Agent 驱动的今日趋势分析报告

**获取真实可播放视频：**

看板中要显示可点击跳转、正常播放的真实今日热门视频，以及真实的播放量、点赞数、互动率，必须：
1. 配置 `TIKHUB_API_TOKEN`（见 2.3）
2. 运行 `python run_all.py` 完成数据抓取
3. 刷新 dashboard

未配置时使用演示数据，播放量/点赞为模拟值，无真实视频链接。

### 3.4 定时任务

**命令：**
```bash
python scheduler.py
```

系统每日 **09:00** 自动执行完整流程。保持终端运行即可。按 `Ctrl+C` 退出。

**修改执行时间：** 编辑 `scheduler.py`，修改：
```python
schedule.every().day.at("09:00").do(job)
```

### 3.5 CSV 数据导入

若需从自有数据源导入视频数据，可：

1. 准备 CSV，需包含列：`video_id`, `title`, `duration`, `play_count`, `like_count`, `comment_count`, `video_url`, `tags` 等（可自定义列名映射）
2. 将文件放入 `data/raw/` 或指定路径
3. 在代码中调用：
   ```python
   from src.data_source.csv_import import import_from_csv
   videos = import_from_csv("data/raw/your_data.csv")
   ```

参考示例格式：`data/raw/sample_tiktok_hot.csv`

### 3.6 员工画像系统（员工数字人）

**用途**：将营销设计公司员工绘制成数字人，AI 生成内容按员工风格输出。

**员工属性**：
- 姓名、岗位、行业经验、内容风格
- 员工 AI 助手类型（搞笑型/剧情型/干货型等）
- 擅长领域
- 自定义 prompt 模板（员工可按岗位规范填写）

**操作流程**：
1. 在「员工画像系统」新增员工，选择 AI 助手类型（如搞笑型创意策划、剧情反转型创意策划）
2. 填写内容风格、擅长领域
3. 可选：填写创意生成、脚本生成的 prompt 模板（可用 `{{topic}}` `{{viral_model}}` `{{idea}}`）
4. 在「创意生成工具」选择该员工，生成内容将匹配其风格
5. 创意历史自动记录，可在员工画像中查看

## 七、常见问题

### Q1：未配置 TikHub Token 时能否使用？

可以。系统会自动加载 `data/raw/sample_tiktok_hot.csv` 作为示例数据，完整流程可照常运行，用于功能演示。

### Q2：LLM 未配置时，创意生成如何工作？

未配置 `DASHSCOPE_API_KEY` 或 `OPENAI_API_KEY` 时，系统使用预设规则模板生成创意与脚本，功能可用但效果较简单。上线测试建议配置通义或 OpenAI。

### Q3：飞书推送失败怎么办？

- 确认 `FEISHU_APP_ID`、`FEISHU_APP_SECRET`、`FEISHU_CHAT_ID` 正确
- 检查应用权限是否包含发送群消息
- 确认机器人已加入目标群聊

### Q4：定时任务在 Windows 上如何常驻运行？

可使用任务计划程序或 `pythonw` 后台运行，也可部署到服务器后使用 `nohup` 或 systemd。

### Q5：如何更换 LLM 供应商？

在 `.env` 中设置 `OPENAI_BASE_URL` 指向兼容 OpenAI 格式的 API 地址，并填写对应的 `OPENAI_API_KEY` 与 `LLM_MODEL`。
