# AI 营销自动化平台 — 实战使用文档

> 从零到功能演示的完整操作指南

---

## 一、文档说明

本文档面向**实际使用与公司演示**场景，按步骤说明如何：

1. 快速完成环境搭建
2. 选择演示模式或真实 API 模式
3. 理解平台 5 大功能模块与企业使用流程
4. 执行完整流程并启动看板
5. 逐页进行功能演示（对应企业流程：输入产品→选择行业→AI 脚本→发布）
6. 上线部署与日常运维

**核心价值**：AI 内容工厂 — 解决营销公司创意、效率、内容生产痛点，约 10 分钟/视频，1 天可生成约 100 条内容。

---

## 二、两种使用模式

| 模式 | 适用场景 | 数据来源 | 配置要求 |
|------|----------|----------|----------|
| **演示模式** | 公司内部演示、培训、产品推介 | 预置 demo CSV（30 条营销向数据） | 仅需 `DEMO_MODE=true` |
| **真实模式** | 正式使用、获取实时 TikTok 数据 | TikHub API 实时抓取 | 需 `TIKHUB_API_TOKEN` + LLM Key |

**建议**：演示优先用演示模式，稳定且无需 API；正式使用再切换真实模式。

## 三、快速上手（5 分钟）

### 3.1 环境要求

- **Python** 3.9 及以上
- **系统** Windows / macOS / Linux

### 3.2 安装依赖

```bash
cd d:\yingxiao
pip install -r requirements.txt
```

### 3.3 演示模式 — 最少配置

1. 复制 `.env.example` 为 `.env`（若尚未创建）

   ```powershell
   copy .env.example .env
   ```

2. 打开 `.env`，设置演示模式：

   ```
   DEMO_MODE=true
   ```

3. （可选）配置 LLM 以获得更好的 AI 创意效果：

   - 通义/百炼：`DASHSCOPE_API_KEY=sk-xxxx`
   - 或 OpenAI：`OPENAI_API_KEY=sk-xxxx`

   未配置时，创意生成会使用规则模板，流程可跑通但效果简化。

### 3.4 执行完整流程

```bash
python run_all.py
```

**预期输出**：
- `[演示模式] 使用预置演示数据，流程稳定、无需 API`
- `抓取完成，共 30 条视频`
- `AI 分析结果已保存`
- `工作流编排 (Trend → Strategy → Script → Score)`
- `趋势入库: X 条 | 脚本入库: Y 条 | 内容分析完成`

### 3.5 启动看板

```bash
streamlit run dashboard.py
```

浏览器会打开 `http://localhost:8501`（端口可能为 8502、8503 等）。

## 四、平台功能模块

平台 UI 规划 **5 个核心模块**，覆盖从趋势分析到内容发布的全流程：

| 模块 | 子功能 | 对应看板 |
|------|--------|----------|
| **1 趋势分析** | 热门视频、爆款结构 | 主页 |
| **2 创意中心** | 生成营销创意 | 创意生成工具（输入产品+选择行业） |
| **3 脚本中心** | AI 脚本、脚本编辑 | 创意生成工具、内容分析 |
| **4 视频生成** | 脚本 → 视频 | Kling API，营销/电商竖屏短视频 |
| **5 数据分析** | 内容分析、传播预测 | 内容分析（分析/评分/编辑/传输） |

## 五、企业使用流程与商业价值

### 5.1 公司员工使用方式

员工按以下 5 步完成一个营销视频：

1. **输入产品** — 输入要推广的产品或服务
2. **选择行业** — 选择所属行业（营销、职场、美食等）
3. **AI 生成脚本** — 系统基于爆款规律自动生成创意与脚本
4. **AI 生成视频** — Kling 文生视频，竖屏 9:16，营销/电商风格
5. **发布** — 推送至目标平台或企业协作（如飞书）

**效率**：约 **10 分钟** 完成一个视频。

### 5.2 商业价值

**营销公司最缺的是**：创意、效率、内容生产。

**本系统可**：1 天生成约 **100 条** 内容。

**核心价值**：**AI 内容工厂** — 全流程自动化，解决创意与产能瓶颈。

## 六、功能演示路线图

按以下顺序演示，可覆盖平台核心能力，并与「平台功能模块」对应。

### 6.1 主页 — 趋势分析（模块 1）

**展示内容**：
- 今日热门视频榜（播放、点赞、互动率）
- 趋势概览（总播放、总点赞、平均时长）
- 视频时长分布
- AI 分析摘要（爆款模型、AI 推荐创意）

**演示话术示例**：
> 「系统自动分析热门视频数据，提取爆款结构，并为营销场景生成推荐创意。」

---

### 6.2 创意中心（模块 2）

**操作**（企业流程）：
1. 侧边栏选择「创意生成工具」
2. **步骤 1**：输入产品/服务（如「智能手环」）
3. **步骤 2**：选择行业（营销、职场、美食等）
4. 可选：选择员工数字人、补充主题
5. 点击「生成创意」

**展示内容**：
- 多个 AI 创意（标题、钩子、标签、建议时长）
- 可展开查看完整脚本（画面、口播、备注）

**演示话术**：
> 「输入产品与行业，AI 基于爆款规律生成创意和脚本，约 10 分钟可完成一条视频的前期策划，支撑 AI 内容工厂的产能目标。」

---

### 6.3 爆款内容库

**操作**：
1. 先点击「同步」（若有）
2. 输入标签如「营销」「职场」检索
3. 查看爆款模型、热门拆解

**展示内容**：
- 爆款内容结构、脚本公式
- 按标签检索的相关拆解

---

### 6.4 员工画像系统

**操作**：
1. 新增员工
2. 填写岗位、内容风格、擅长领域
3. 返回创意生成工具，选择该员工后生成

**展示内容**：
- 员工数字人管理
- 创意风格可定制
- 创意历史记录

---

### 6.5 Agent 与 RAG

**操作**：
1. 在 Agent 对话框输入「分析今日热门趋势」
2. Agent 调用工具并返回分析
3. RAG 同步后，输入「搞笑创意」检索

**展示内容**：
- Agent 自动调用热门榜、爆款模型、RAG
- RAG 语义检索

---

### 6.6 内容分析（模块 5）

**操作**：
1. 侧边栏选择「内容分析」
2. 待分析：创意中心/历史发送的脚本；工作流脚本库：`run_all.py` 生成的脚本
3. 点击「分析评分」获取创意/营销价值/传播潜力；可编辑脚本后保存
4. 点击「传输给视频生成」将脚本传到视频生成页面

**展示内容**：
- 待分析（创意中心）、工作流脚本库、已传输视频生成
- AI 分析：创意评分、营销价值、传播潜力

> **说明**：脚本在内容分析中完成评分、编辑后，可传输到「视频生成」。视频生成已对接 Kling API，需配置 `KLING_ACCESS_KEY`+`KLING_SECRET_KEY`（或 `KLING_API_KEY`，https://klingapi.com）。

## 七、从演示到上线

### 7.1 切换到真实模式

1. 获取 TikHub API Token：<https://tikhub.io> 注册并创建 Token  
2. 获取 LLM Key：阿里云百炼或 OpenAI  
3. 修改 `.env`：

   ```
   DEMO_MODE=false
   TIKHUB_API_TOKEN=sk-xxxxxxxx
   DASHSCOPE_API_KEY=sk-xxxxxxxx
   KLING_ACCESS_KEY=xxx        # 视频生成（可选）
   KLING_SECRET_KEY=xxx        # 与上面成对，https://klingapi.com
   ```

4. 中国大陆用户如访问 TikHub 异常，可增加：

   ```
   TIKHUB_API_BASE=https://api.tikhub.dev
   ```

### 7.2 上线前验证

| 步骤 | 命令 | 验收点 |
|------|------|--------|
| 1 | `python run_all.py` | 无报错，输出热门榜、AI 分析、工作流完成 |
| 2 | `streamlit run dashboard.py` | 打开 localhost |
| 3 | 主页 | 有真实视频链接（非 demo）、可点击 |
| 4 | 创意生成 | 生成自然、专业的创意和脚本 |
| 5 | Agent | 输入「分析今日热门趋势」有分析结果 |
| 6 | 内容分析 | 有脚本及分析/评分 |
| 7 | 视频生成 | 配置 KLING_API_KEY 后可生成视频 |

### 7.3 定时任务（可选）

```bash
python scheduler.py
```

每日 09:00 自动执行 `run_all.py`。修改时间可编辑 `scheduler.py` 中 `schedule.every().day.at("09:00")`。

### 7.4 飞书推送（可选）

1. 登录 [飞书开放平台](https://open.feishu.cn/) 创建应用  
2. 开通 `im:message` 等权限  
3. 将机器人拉入群聊，获取群 `chat_id`  
4. 在 `.env` 中配置：

   ```
   FEISHU_APP_ID=xxx
   FEISHU_APP_SECRET=xxx
   FEISHU_CHAT_ID=xxx
   ```

5. 运行 `run_all.py` 后会自动推送日报和创意

## 八、常见问题

### Q1：演示时没有数据或报错？

- 确认已执行 `python run_all.py` 且成功
- 确认 `DEMO_MODE=true` 且 `data/raw/demo_tiktok_hot.csv` 存在
- 检查 `data/processed/` 下是否有 `hot_rank_*.csv` 和 `ai_analysis_*.json`

### Q2：创意生成效果很模板化？

- 配置 `DASHSCOPE_API_KEY` 或 `OPENAI_API_KEY`
- 重新运行 `run_all.py` 后刷新看板

### Q3：TikHub 返回 403？

- 检查 Token 是否正确、是否过期
- 中国大陆可尝试 `TIKHUB_API_BASE=https://api.tikhub.dev`

### Q4：内容分析无工作流脚本？

- 需先运行 `run_all.py`（包含工作流）
- 确认 `.env` 中 `USE_WORKFLOW=true`（默认开启）
- 创意中心生成的脚本可直接「发送到内容分析」，无需工作流

### Q5：视频生成提示未配置？

- 注册 [Kling API](https://klingapi.com) 获取密钥
- **双密钥**：`KLING_ACCESS_KEY` + `KLING_SECRET_KEY`（访问密钥+密钥，推荐）
- **单密钥**：`KLING_API_KEY`（若平台提供 Bearer 方式）
- 需安装 `PyJWT`（双密钥时用 JWT 签名）

### Q6：看板端口被占用？

Streamlit 会自动尝试 8501、8502 等，或手动指定：

```bash
streamlit run dashboard.py --server.port 8505
```

## 九、视频生成（Kling）

流程：创意中心 → 内容分析 → 传输给视频生成 → 点击「生成视频」→ 等待约 30–60 秒 → 播放/下载

- **规格**：竖屏 9:16，5 秒，营销/电商风格
- **模型**：kling-v2.5-turbo（快速）
