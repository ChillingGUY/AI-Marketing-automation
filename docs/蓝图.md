# AI 营销自动化平台 — 蓝图

> **设计目标**：把内容策划、脚本生成、视频制作、趋势分析等流程自动化，并通过企业协作平台分发给员工使用。

---

## 一、愿景与价值

| 维度 | 目标 |
|------|------|
| **创意** | AI 辅助内容策划与创意生成 |
| **效率** | 脚本、视频自动化生产，约 10 分钟/视频 |
| **产能** | 1 天可生成约 100 条内容 |
| **分发** | 通过飞书等企业协作平台自动推送 |

**核心定位**：AI 内容工厂 — 端到端自动化、企业协作分发。

---

## 二、技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                    Frontend (Streamlit / 未来: Next.js)       │
├─────────────────────────────────────────────────────────────┤
│                    Backend API (未来: FastAPI)                │
├─────────────────────────────────────────────────────────────┤
│              Agent Orchestrator (LangGraph / 自研编排)         │
│   Trend → Strategy → Script → Video → Edit → Score → Publish │
├──────────────────────────────────┬──────────────────────────┤
│   LLM Engine (OpenAI/通义)       │   Tools Layer            │
│   Prompt / RAG / ChromaDB        │   视频API / 数据API       │
├──────────────────────────────────┴──────────────────────────┤
│   Data Layer: CSV → SQLite → PostgreSQL (演进)               │
├─────────────────────────────────────────────────────────────┤
│   Message Queue (未来: Redis / Celery) 用于视频生成等长任务    │
└─────────────────────────────────────────────────────────────┘
```

## 三、八阶段路线图

### Phase 1：基础数据与分析（当前已完成 ✅）

| 模块 | 内容 | 状态 |
|------|------|------|
| 数据采集 | TikTok 热门视频、CSV 导入 | ✅ |
| ETL | 清洗、排名、互动率计算 | ✅ |
| AI 分析 | 爆款结构、创意生成、脚本输出 | ✅ |
| 知识库 | 爆款库、RAG 向量检索 | ✅ |
| 推送 | 飞书日报、创意推送 | ✅ |
| 看板 | Streamlit 趋势、创意、Agent、RAG | ✅ |

### Phase 2：Agent 编排与结构化存储（下一阶段）

| 模块 | 内容 |
|------|------|
| 工作流编排 | LangGraph 或自研 StateGraph，串联 Trend→Strategy→Script |
| 数据库 | SQLite/PostgreSQL：trends、scripts、videos、agents_log |
| Agent 抽象 | 统一 Agent 接口、输入输出 schema |
| 内容评分 | 创意评分、营销价值、传播潜力（LLM 评估） |

### Phase 3：脚本中心与视频生成接口

| 模块 | 内容 |
|------|------|
| 脚本中心 | 脚本 CRUD、编辑、版本管理 |
| 视频 API 对接 | Runway / Kling 等接口封装（可选，可 mock） |
| 脚本→视频 | 将脚本提交到视频生成队列（异步占位） |

### Phase 4：剪辑与发布增强

| 模块 | 内容 |
|------|------|
| 剪辑 | ffmpeg/moviepy 封装：字幕、BGM、节奏剪辑 |
| 发布 | 飞书视频推送、运营建议自动发送 |
| 数据复盘 | 播放数据采集、下一轮策略建议 |

### Phase 5：企业级与规模化

| 模块 | 内容 |
|------|------|
| Backend | FastAPI 独立 API 服务 |
| 消息队列 | Redis + Celery 处理长任务 |
| 前端 | Next.js / Vue 企业端（可选） |
| 多租户 | 按企业/团队隔离数据 |
